PROMPT 4 — Контрольные точки и проверяемые гипотезы роста (вход для PROMPT 5)
Роль: ты — стратег роста EVIA Studio.
Цель PROMPT 4:
Сформировать и приоритизировать проверяемые гипотезы роста и контрольные точки, которые дадут доказательства:
наличия коммерческого спроса (лиды / диалоги с ЛПР / первые подтверждения намерений)
и/или потенциала воспроизводимого канала (стабильный механизм получения сделок)
Важно (обязательное разъяснение):
PROMPT 4 не задаёт отдельный период и не объявляет “спринт”.
PROMPT 4 не принимает решение о запуске работ EVIA.
PROMPT 4 создаёт доказательную основу и контрольные точки, которые затем используются в PROMPT 5 для решения: берём ответственность или нет.
PROMPT 4 существует не ради “идей”, а ради ответа:
Есть ли подтверждённый потенциал совместной работы с EVIA через воспроизводимое привлечение ЛПР и управляемую экономику?

INPUT (используй выводы PROMPT 1–3 и 3.5–3.7)
PROMPT 1 — Пакет фактов о компании (проверка реальности)
PROMPT 2 — Клиент, рынок и ёмкость
PROMPT 3 — Конкуренты, отличие, свободные зоны, пригодность каналов
PROMPT 3.5 — Выбранная модель роста + ограничения поставки + условия остановки
PROMPT 3.6 — Ключевое ценностное предложение + «пакет доказательств»/пример результата
PROMPT 3.7 — Топ-3 автоматизации/ИИ с измеримым эффектом:
рост конверсии закрытия (измеряется через письма о намерениях / депозиты)
трудозатраты на клиента (измеряются тайм-трекингом)
доля превышений лимитов нагрузки
доля лидов с полными данными

ОБЯЗАТЕЛЬНОЕ ПРАВИЛО: контекст должен быть перенесён внутрь документа
Перед тем как генерировать гипотезы, ты обязан:
извлечь ключевые факты и выводы из PROMPT 1–3 и 3.5–3.7
перенести их внутрь текущего ответа
сделать PROMPT 4 автономным документом
Запрещено писать: «см. PROMPT 2», «как указано в PROMPT 3» и т.п.

1) Краткий контекст проекта (1 экран)
Сначала выдай короткий контекст (в 10–15 строк):
что продаём
кому продаём
в чём отличие
какие каналы считаем наиболее вероятными
какие ограничения и риски уже известны
что является “узким местом” (канал / продукт / продажи / экономика / ресурсы)

2) Минимальная проверка экономической пригодности (обязательно)
Собери минимальную проверку, чтобы гипотезы не были “в вакууме”:
средний чек (факт/оценка)
маржа (факт/оценка)
целевая экономика привлечения (в логике окупаемости)
ожидаемая конверсия по воронке (явно пометить: факт или допущение)
Результат: короткий вывод:
экономика “в принципе пригодна” / “сомнительна” / “не пригодна”, и почему.

3) Проверка способности поставки (обязательно)
Сделай расчёт и вывод:
трудозатраты на 1 клиента (факт или измеряемый диапазон)
максимальное количество активных клиентов при текущей доступной нагрузке
ожидаемая нагрузка при вероятном потоке клиентов
запас по нагрузке = доступная мощность / ожидаемая нагрузка
Правило риска:
если запас по нагрузке < 1.2 — это критический риск, который должен быть явно помечен для PROMPT 5.
(Здесь ты не принимаешь решение STOP, ты фиксируешь риск как красный флаг.)

4) Список проверяемых гипотез роста (обязательно)
Сформируй 8–12 гипотез, каждая строго в формате:
Гипотеза: …
Канал: …
Механика: как именно получим контакты/диалоги/встречи
Кому пишем/выходим: тип ЛПР / сегмент
Почему должно сработать: привязка к фактам из контекста
Что считаем доказательством: чёткие измеримые признаки
Срок/объём усилия: оценка в действиях (не “дни”)
Риски/ограничения: что может сломать гипотезу

5) Приоритизация гипотез (обязательно)
Отранжируй гипотезы по трём критериям (по 10-балльной шкале):
вероятность результата
скорость получения доказательств
стоимость/сложность выполнения
Дай итоговый список ТОП-3 и объясни почему именно они.

6) Контрольные точки (обязательно)
Собери контрольные точки как “ворота качества”:
что должно быть получено, чтобы усиливать канал
что должно быть получено, чтобы остановиться или поменять подход
какие данные должны быть собраны обязательно (чтобы избежать самообмана)

Выход PROMPT 4
Контекст проекта (коротко)
Экономическая пригодность (кратко)
Проверка способности поставки (и красные флаги)
8–12 проверяемых гипотез в заданном формате
ТОП-3 гипотезы + обоснование
Контрольные точки и правила остановки/изменения

Строгое правило качества
Любое утверждение помечай как:
ФАКТ (если есть данные)
ДОПУЩЕНИЕ (если оценка)
ГИПОТЕЗА (если требует проверки)
