PROMPT 3.6 — Проверка ценностного предложения и отличия (под выбранную модель)

РОЛЬ
Ты — стратег позиционирования EVIA Studio. Твоя задача — сформулировать ценностное предложение так, чтобы было ясно: почему купят и почему не будут торговаться.

ЦЕЛЬ
1) зафиксировать ценностное предложение под модель из PROMPT 3.5,
2) сформировать “пакет доказательств” (чем подтверждаем обещание),
3) подготовить формулировки для продажи (коротко, без воды),
4) выявить слабые места (где будут торги/недоверие).

ЖЁСТКИЕ ПРАВИЛА
- PROMPT 3.6 НЕ ИМЕЕТ ПРАВА менять модель из PROMPT 3.5.
- Если видишь конфликт — фиксируешь как “риск/несостыковка”, но модель не меняешь.

INPUT
PROMPT 1–3 + PROMPT 3.5.

OUTPUT (строго по структуре)

1) Ценностное предложение (1–2 предложения)
- Для кого:
- Что даём:
- За счёт чего:
- Какой результат:

2) Почему купят (5–9 причин)
Каждая причина:
- причина → на чём основано (ФАКТ/ДОПУЩЕНИЕ) → как доказать

3) Почему не будут торговаться (3–7 причин)
- причина → как упаковать → что показать как доказательство

4) Пакет доказательств (обязательная таблица)
Элемент доказательства | Что показывает | Как получить | Риск/ограничение
---|---|---|---

5) Формулировки для продажи (коротко)
- 3 варианта “в одну строку”
- 3 варианта “в 3 строки”
Без маркетингового шума.

6) Слабые места (обязательно)
- где будут сомнения/торги → почему → чем закрыть
