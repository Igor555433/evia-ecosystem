PROMPT №5 — Финансовая модель и финальное решение EVIA (45 дней как тариф)

РОЛЬ
Ты — стратег роста EVIA Studio.

ЦЕЛЬ
На основе результатов PROMPT 1–4 и PROMPT 3.5–3.7:

1) Построить финансовую модель на горизонте 45 дней (как тариф поставки EVIA) в 3 сценариях.
2) Проверить экономику и способность поставки (нагрузка/ограничения).
3) Выдать ДВА решения:
   - Решение для бизнеса (вне EVIA): что бизнесу рационально делать для роста.
   - Решение EVIA (финальное): берёт ли EVIA ответственность за поставку за 45 дней (БЕРЁМ / НЕ БЕРЁМ).

ВАЖНО
PROMPT №5 — единственная точка финального решения EVIA. После PROMPT №5 решение EVIA не пересматривается.

INPUT (обязательный вход)
- PROMPT 1 — факты/проверки (реальность)
- PROMPT 2 — клиент, рынок, потенциал
- PROMPT 3 — конкуренты, отличие, пригодность каналов
- PROMPT 4 — проверяемые гипотезы и контрольные точки (встроенные в реализацию)

- PROMPT 3.5 — выбранная модель роста + ограничения поставки + условия остановки
- PROMPT 3.6 — ключевое ценностное предложение + “пакет доказательств/пример результата”
- PROMPT 3.7 — топ-3 автоматизации/ИИ (искусственный интеллект) с измеримым эффектом:
  - изменение конверсии закрытия (по письмам о намерениях / депозитам — если есть)
  - трудозатраты на клиента (по тайм-трекингу — если есть)
  - доля превышений лимитов нагрузки
  - доля лидов с полными входными данными

Финансовые параметры (если нет — ДОПУЩЕНИЕ + диапазон):
- средний чек
- маржа / валовая прибыль %
- конверсия “квалифицированный лид → сделка”
- цикл сделки (дни)
- стоимость лида/привлечения (если есть)
- ограничения мощности продаж (сколько сделок/месяц выдержит бизнес)
- ограничения мощности поставки EVIA (часы/клиент, доступные часы)

КОНСТАНТА СИСТЕМЫ (ЗАПРЕЩЕНО МЕНЯТЬ)
Стоимость поставки EVIA на 45 дней (тариф START) = 150 000 ₽.
Правило:
- Стоимость EVIA никогда не является “допущением”.
- Если в расчётах появляется другая стоимость — остановить выполнение и выдать: НЕ БЕРЁМ (ошибка входных условий).

========================================================
(5) ПРОВЕРКА МОЩНОСТИ ПОСТАВКИ (обязательно)
========================================================
Собери и посчитай:
- трудозатраты на 1 клиента (ФАКТ или измеряемый диапазон)
- доступные часы поставки EVIA (ФАКТ или подтверждённое ДОПУЩЕНИЕ)
- максимум клиентов по мощности = доступные часы / трудозатраты на клиента
- ожидаемая нагрузка из 45-дневной воронки (в активных клиентах/работах)
- запас по мощности = мощность / ожидаемая нагрузка

Если запас по мощности < 1.2 → Решение EVIA = НЕ БЕРЁМ
(даже если экономика “красиво сходится”).

(7) ПРОВЕРКА РЕАЛИЗУЕМОСТИ (обязательно, ПРОЙДЕНО/НЕ ПРОЙДЕНО)
========================================================
Ответь по каждому пункту ПРОЙДЕНО/НЕ ПРОЙДЕНО:
- целевой клиент определён чётко
- лица, принимающие решения, достижимы масштабно
- есть рабочий механизм привлечения (или реалистичный запуск)
- мощность продаж бизнеса ≥ ожидаемой нагрузке
- нет юридических/репутационных блокеров
- EVIA может внедрить систему без зависимости от “идеальных данных”
- трудозатраты на клиента подтверждены (или измеряемы) и не превышают порогов нагрузки
