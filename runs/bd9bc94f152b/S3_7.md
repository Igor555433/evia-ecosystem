# TEAM & OPERATING MODEL для проекта Alpha AI

## 1. Роли и ответственность
- **Проектный менеджер**: отвечает за общее управление проектом, координацию команды и взаимодействие с заказчиком.
- **Аналитик**: отвечает за анализ целевой аудитории и формирование ICP.
- **Frontend-разработчик**: отвечает за разработку пользовательского интерфейса и его тестирование.
- **Backend-разработчик**: отвечает за разработку API и базы данных.
- **Инженер по интеграции**: отвечает за интеграцию систем и взаимодействие с аутсорсинговой компанией.
- **Тестировщик**: отвечает за тестирование системы перед запуском.

## 2. Ритм коммуникаций и отчётности
- **Еженедельные встречи**: команда собирается раз в неделю для обсуждения текущего статуса, проблем и планов на следующую неделю.
- **Отчёты**: проектный менеджер предоставляет отчёты о ходе выполнения проекта каждые две недели.
- **Критические контрольные точки**: по завершении каждого этапа (по графику) проводится встреча для оценки результатов и принятия решений о дальнейшем движении.

## 3. Требуемые входы от заказчика
- **Данные о целевой аудитории**: информация, необходимая для анализа и определения ICP.
- **Технические требования**: спецификации для разработки Frontend и Backend.
- **Контакты аутсорсинговой компании**: для организации интеграции систем.
- **Бюджетные ограничения**: информация о финансовых рамках проекта.

## 4. Ограничения, блокирующие старт
- **Отсутствие данных о целевой аудитории**: без информации о целевой аудитории невозможно провести анализ и определить ICP.
- **Неопределённые технические требования**: отсутствие четких спецификаций может привести к задержкам в разработке.
- **Проблемы с доступом к аутсорсинговой компании**: если не удастся установить контакт, это затруднит интеграцию систем.
